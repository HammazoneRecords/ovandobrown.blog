## PART 5 — TECHNICAL SEO

### 5A — THE SPA BLOCKER (Priority 1 — Fix Before Everything Else)

**Problem:** whatissolob.com is a pure client-side React SPA (Vite + react-router-dom v7). Google lands on it and gets an almost-empty HTML document. The entire site content renders in JavaScript. This means:
- The gate descriptions are not indexed
- The philosophical vocabulary is not indexed
- The Volume Zero content is not indexed
- Even the basic "what is Solobility" answer is not indexed

**Google's crawler does eventually execute JavaScript**, but it is slow, unreliable, and does not crawl all routes of an SPA the way it crawls static HTML. The site has been live for months with essentially zero organic surface.

**Fix options (in order of effort):**

| Option | Effort | Effect |
|---|---|---|
| **`vite-ssg` (Static Site Generation)** | Medium — 1–2 days | Pre-renders all routes to static HTML at build time. Best option. |
| **`react-snap`** | Low — half a day | Crawls the app in headless Chrome, snapshots HTML. Good quick fix. |
| Add static landing pages alongside SPA | Medium | Create `/about`, `/gates/*`, `/glossary` as separate static HTML pages |
| Move to Next.js (SSR/SSG) | High — full rebuild | Best long-term but too much work now |

**Recommendation:** `vite-ssg` — it integrates with Vite, supports react-router-dom, and outputs a fully pre-rendered static site. The existing app code barely changes. Build output becomes crawlable HTML for every route.

Install: `npm install vite-ssg`
Add to `vite.config.ts`:
```ts
import { ViteSSG } from 'vite-ssg'
// wrap app export with ViteSSG
```

**This is the single highest-impact SEO action available. Without it, everything else is irrelevant.**
